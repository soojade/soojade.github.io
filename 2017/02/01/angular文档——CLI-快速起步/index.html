CLI 快速起步

使用 CLI 工具快速构建 Angular 应用。

相比手动的做一切事情，好的工具使应用开发更快、更易于维护。

Angular-CLI 是一个命令行工具，使用它可以创建项目、添加文件，并执行各种开发中的任务，例如——测试、打包、部署。

在本节，我们的目标是使用 Angular-CLI 并遵循风格指南的推荐，构建和运行一个超级简单的 Typescript版的 Angular 应用，这将有益于每一个 Angular 项目。

本章结束后，你会对使用 CLI 开发有一个基本的理解,并且有为这些文档案例和我们的真实应用的项目基础。

要完成目标，包含下面几个主要的步骤：

  1. 配置开发环境
  2. 新建项目及应用架构
  3. 启动应用服务器
  4. 编辑应用

第一步：配置开发环境

在你做任何事之前，需要配置我们的开发环境。

安装 Node.js 和 npm（假如你的电脑还没有安装）。

通过在命终端/命令行运行node -v和npm -v确保你运行的版本 node 最低4.x.x，npm 最低3.x.x。旧版本会产生错误，但新版本不会。

第三步：启动应用服务器

进入项目目录并启动服务器。

ng serve命令启动服务器，并监视文件，当文件发生改变后重新编译应用。

打开浏览器，输入http://localhost:4200/，显示欢迎信息：app works!。

接下来做什么？

你可以在“Hello，World”应用中做任何你想做的事情。

你已经学习了英雄之旅教程并且构建了一个小型应用，这是使用 Angular 构建应用的伟大的演示。

或者你可以花费一段时间来学习新项目中的文件。

项目文件回顾

Angular-CLI 项目是快速构建实验和企业解决方案的基础。

我们查看的第一个文件应该是README.md。它有一些如何使用 CLI 命令的基础信息。无论何时你想要知道更多的关于 Angu-CLI 如何工作的信息，一定要访问Angular-CLI仓库和Wiki。

一些生成的文件可能是你不熟悉的。

src 目录

你的应用存放在src目录下。所有的 Angular 组件、模板、样式、图片和任何你的应用需要的其他东西都在这里。这个目录之外的任何文件都是为了支持构建你的应用。

文件 用途
app/app.component.{ts,html,css,spec.ts} 定义了AppComponent相关的HTML模板、css样式和单元测试。随着应用程序的进化，它将成为嵌套组件树的根组件。
app/app.module.ts 定义AppModule，根模块告诉Angular如何配置应用程序。现在它只声明了AppComponent。之后会有更多的组件声明。
assets/* 一个可以存放图片和构建应用程序时需要批量复制的任意东西的目录。
environments/* 这个目录包含一个为每个目标环境的配置文件，每一个导出的简单的配置变量用到你的应用中。当你构建应用时，文件会被动态的替换。你可能在开发时使用与生产时不同的后端API。或许可能不同的解析令牌。甚至可能是一些模拟服务。无论哪种方式，CLI 都包括了。
favicon.ico 每个网站都想在书签栏看起来好看。开始创建你自己的 Angular ico。
index.html 当有人访问你网站时的主页。大多数时候你永远也不需要编辑它。当构建应用时，CLI会自动地添加所有的 js 和css文件，所以你从来都不需要在这里手动的添加任何的<script>或<link>标签
main.ts 应用的主入口。使用JIT编译器编译应用并引导应用在浏览器中运行。你也可以通过给ng build或ng serve传入--aot参数，而不需要改变任何代码来使用AoT编译器。
polyfills.ts 不同的浏览器对web标准的支持有不同的级别。Polyfills帮助标准化这些不同。你可以非常安全的使用core-js和zone.js，但是一定要查看浏览器支持来获取更多的信息。
style.css 你的全局样式在这里。大多数时候你会需要在你的组件中使用局部样式以易于维护，但影响整个应用的样式需要在一个主要的地方。
test.ts 这是单元测试的主入口。它有一些你可能不熟悉的自定义的配置，但你不需要去编辑。
tsconfig.json Typescript编译器的配置文件

根目录

src/目录仅仅是项目根目录的一项。其他文件帮助你构建、测试、维护、文档和部署应用。
这些文件在根目录中挨着src/。

文件 用途
e2e/* e2e/中存放着端到端测试。它们不在src/里，是因为e2e测试实际上是一个为了测试你的应用的独立的应用。这就是为什么他们甚至有自己的tsconfig.json。
node_modules/... Node.js创建这个目录并把package.json中所列出的第三方模块放到里面。
.editorconfig 你的编辑器的简单配置，以确保每个使用你项目的人有相同的基本配置。大多数编辑器提供.editorconfig文件，查看 http://editorconfig.org 获取更多信息。
.gitignore git的配置来确保自动生成的文件不被提交到源码控制中。
angular-cli.json Angular-CLI的配置。在这个文件中你可以设置几个默认值，也可以配置当项目构建时所包含的文件。如果你想了解更多，查看官方文档。
karma.conf.js Karma运行时的单元测试配置，当运行ng test时使用。
package.json 你的项目使用的第三方包的npm配置清单。你也可以在这里添加你自己的自定义脚本。
`README.md 你的应用的基本文档，预先填写了使用 CLI 的命令信息。确保增加项目文档强化它，以便于任何查看你仓库的人都可以构建你的应用。
tslint.json TSlint连同Codelyzer的配置，当运行ng lint时使用。Linting 可以帮助你保持代码风格始终如一。

下一步

如果你是Angular新手，我们推荐继续学习路径。

你可以跳过“setup”章节，因为你已经使用了Angular-cli。
