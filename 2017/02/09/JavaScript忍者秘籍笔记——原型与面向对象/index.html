JavaScript 忍者秘籍笔记——原型与面向对象

JavaScript
  • JavaScript忍者秘籍笔记

第六章 原型与面向对象

原型虽然是定义对象的一种很方便的方式，但它的本质依然是函数特性。

实例化和原型

所有的函数在初始化的时候都有一个 prototype 属性，该属性的初始值是一个空对象。只有函数在作为构造器的时候，prototype 属性才会发挥更大的作用。

我们并没有在构造器里显式地做任何事情，通过将swingSword()方法添加到构造器的prototype属性上，swingSword()方法就可以附加到新对象上了。

使用new操作符将函数作为构造器进行调用的时候，其上下文被定义为新对象实例。这意味着，除了通过原型给函数附加属性的形式外，还可以在构造器函数内通过this参数初始化。初始化优先级如下：

  1. 在构造器函数内给对象实例添加的属性。
  2. 通过原型给对象实例添加的属性。

在构造器内的绑定操作优先级永远都高于原型上的绑定操作。因为构造器的this上下文指向的是实例自身，所以可以在构造器内对核心内容执行初始化。

对象属性引用查找的一般过程：

  1. 在引用对象的一个属性时，首先检查该对象本身是否拥有该属性，如果有，则直接返回；
  2. 如果没有，再查看对象的原型(prototype)，检查该原型上是否有所要的属性，如果有直接返回；
  3. 如果没有，该值是 undefined。

JavaScript 中每个对象，都有一个名为 constructor 的隐式属性，该属性引用的是创建该对象的构造器。由于 prototype 是构造器的一个属性，所以每个对象都有一个方式可以找到自己的原型。

constructor属性作为创建该对象的原始函数的引用，被添加到所有的实例上。利用该属性，可以验证实例的起源，就像instanceof操作符一样。由于该属性只是原始构造器的一个引用，所以可以使用该引用再次实例化一个新对象。

尽管对象的constructor属性可以被修改，这样做没有任何直接或很明显的目的，只是告知对象在哪里被创建的。如果constructor属性被覆盖，那么创建该对象的原始函数就丢失了。

继承与原型链

instanceof操作符还有另一个功能，利用它作为对象继承的一种形式。通过执行instanceof操作，可以判断函数是否继承了其原型链中任何对象的功能。

不要这样：Ninja.prototype = Person.prototype，这样做的话，Ninja原型上的任何修改都会影响到Person的原型，因为他们是同一个对象。

HTML DOM 原型

所有的 DOM 元素都继承于HTMLElement构造器。通过访问HTMLElement的原型，浏览器可以为我们提供扩展任意 HTML 节点的能力。

注意：HTML元素不可以直接通过其构造器进行实例化。
