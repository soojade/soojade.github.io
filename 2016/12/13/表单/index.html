表单

表单是商业应用的支柱，我们用它来执行登录、求助、下单、预订机票、安排会议，以及不计其数的其它数据录入任务。

任何经验丰富的 Web 开发人员都能使用适当的标签拼凑出 HTML 表单。 但是，要想做出具有贴心的数据输入体验的表单， 引导用户明晰、高效地完成表单背后的工作流程，挑战就大多了。

老实说，这当中所需要的设计技能超出了本章的范围。

双向数据绑定、变更跟踪、有效性验证和错误处理等功能离不开框架的支持。 本章将介绍 Angular 表单相关的内容。

下面，从零开始，一步一步构建出一个简单的表单。在这个过程中，我们将学会如何：

  • 怎样使用组件和模板构建 Angular 表单
  • ngModel双向数据绑定语法，用于读取和写入输入控件的值
  • ngControl指令跟踪表单控件的状态变化和有效性
  • 怎样使用ngControl添加给表单控件的特殊 CSS 类，提供强大的视觉反馈
  • 怎样向用户显示验证错误提示，以及启用/禁用表单控件
  • 怎样利用模板引用变量在控件之间共享信息

模板驱动的表单

通常，我们使用 Angular 模板语法编写模板，结合本章所描述的表单专用指令和技术来构建表单。

这不是创建表单的唯一方式，本章中只介绍模板驱动的表单。

利用 Angular 模板，可以构建几乎所有表单——登录表单、联系人表单…… 以及任何的商务表单。 可以创造性的摆放各种控件、把它们绑定到数据、指定校验规则、显示校验错误、有条件的禁用或 启用特定的控件、触发内置的视觉反馈等等，不胜枚举。

它用起来很简单，这是因为 Angular 处理了大多数重复、单调的任务，这让我们可以不必亲自操刀、身陷其中。

只需要前面章节中学过的概念，就可以完全理解这个组件：

  1. 代码从 Angular 库导入一套标准的符号。
  2. @Component选择器hero-form表示可以用<hero-form>标签把这个表单放进父模板。
  3. moduleId属性设置了相对模块的基地址，例如templateUrl。
  4. templateUrl属性指向一个独立的名为hero-form.component.html的 HTML 模板文件。
  5. 为model和powers定义了供演示用的假数据。 将来，可以注入服务来获取和保存真实数据， 或者暴露这些属性为输入与输出属性，绑定到父组件上。 我们现在先不关心这些，因为这些未来的变化不会影响到表单。
  6. 在最后增加diagnostic属性，它返回这个模型的字符串描述。在开发过程中，它会让我们看到我们所做的事情，给自己留个备注，最后离开时提醒我们丢掉它。

为什么在这组件文件中不是内联模板呢？

当模板很短时内联模板是很好的，但大多数表单模板并不短。Dart 文件通常不是写（或读）大型 HTML 的最好的地方，而且没有几个编辑器能对混写的 HTML 和代码提供足够的帮助。内容清晰、目标明确的短文件也是很好的。

我们做了一个好的选择把 HTML 模板放到别处。让我们编写它。

本节是对[(ngModel)]的深入剖析，它是可选的。不感兴趣？跳过它！

从绑定语法中的[()]开始出发，是一个很好的线索。

属性绑定使数据流从模型中流向屏幕上的目标属性。我们通过把属性名包裹在方括号[]中来标记出目标属性。 这是从模型到视图的单向数据绑定。

事件绑定使数据流从屏幕上的目标属性流向模型。我们通过把属性名包裹在圆括号()中来标记出目标属性。这是从视图到模型反向的单向数据绑定。

难怪 Angular 使用了组合的标点[()]，来标记出双向数据绑定和双向数据流。

事实上，可以把NgModel绑定拆成两个独立的模式，就像下面重写的 Name的<input>绑定一样：

这个属性绑定看起来很眼熟，但事件绑定看起来有点怪。

ngModelChange指定NgModel指令的事件属性。 当 Angular 在表单中看到[(x)]的绑定目标时， 它会期待这个x指令有一个名为x的输入属性，和一个名为xChange的输出属性。

模板表达式中的另一个古怪之处是model.name = $event。 之前看到的$event对象来自 DOM 事件。 但ngModelChange属性不会生产生 DOM 事件；它是Angular EventEmitter类型的属性，当它触发时， 它返回的是输入框的值——这正是我们应该赋值给模型name属性的值。

很高兴知道这些，但是这样现实吗？[(ngModel)]通常是我们想用的，但当事件处理器需要做一些特别的事情，例如合并或限制按键频率时，我们可能会拆分绑定。

通过 ngControl 跟踪修改状态与有效性验证

表单不仅是关于数据绑定的。我们还想知道表单中各个控件的状态。NgControl指令为我们保留了对控件状态的追踪。

NgControl是NgForm指令家庭中的一员，只能被应用在<form>标签内部的控件。

我们的应用可以访问一个NgControl的实例，用户是否碰过此控件，值是否改变过，或值是否有效。

NgControl不仅仅追踪状态；它还使用特定的 Angular CSS 类来更新控件，例如ng-valid或ng-invalid。我们可以使用这些类名来修改控件的外观，显示或隐藏消息。

NgForm 指令

回想之前的部分ngControl作为“name”通过NgForm注册到输入框。

我们我们不明确的添加NgForm指令。Angular 隐式的添加到整个<form>元素。

NgForm指令为<form>元素扩充了额外的特性。它收集指令（被ngControl指令识别的元素）并且监视它们的属性变化，包括有效性。它还有自己的valid属性，只有当其中所有控件都有效时，它才有效。

在本例中，我们把“name”控件抛出它的控件集合，并且把它分配给一个模板引用变量以便我们能够访问控件的属性——比如控件自身的valid属性。

总结

本章讨论的 Angular 表单技术利用了下列框架特性来支持数据修改、验证和更多操作：

  • Angular HTML 表单模板。
  • 带有Component装饰器(Dart中应该是annotation)的表单组件类。
  • 用来处理表单提交的ngSubmit指令。
  • 模板引用变量，例如#heroForm、#name、#power和#spy。
  • 用于双向数据绑定的ngModel指令。
  • 用于确认和追踪表单元素变化的ngControl指令。
  • 指向 input 控件的引用变量上的valid属性，可用于检查控件是否有效、是否显示/隐藏错误信息。
  • 当表单无效时，使用属性绑定使提交按钮不可用。
  • 定制 CSS 类来给用户提供无效控件的视觉反馈。
